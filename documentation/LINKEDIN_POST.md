**The Solution:**

**Here's what happened:**

**1. The Feed Algorithm Challenge**
I needed to show problems in order of importance, not just chronologically. So I built a weighted scoring system:
- Upvotes × 0.4 (community cares)
- Priority × 0.3 (how urgent)
- Recency × 0.2 (time sensitivity)
- Status × 0.1 (actionability)

This is weighted graph algorithms - but applied to content ranking. The result? Critical issues naturally rise to the top.

**2. The Route Optimization Problem**
Field workers had 10+ locations to visit. Random order = wasted time. So I implemented TSP (Traveling Salesperson Problem) - the same algorithm we study as "NP-hard" becomes a practical tool that saves hours and fuel.

**3. The Speed Challenge**
Users complained about slow loading. I implemented two-tier caching:
- In-memory cache (fast, 5-min expiry)
- Persistent cache (offline capability)
- LRU eviction (memory management)

Data structures principles = instant app experience.

**4. The AI Integration**
Used Google Gemini AI for automatic problem classification, orchestrated with LangChain. This isn't just calling an API - it's building intelligent workflows that route problems correctly without human intervention.
